[목표]
컨디션노트(Condition Note) 앱의 “AI 추천 → YouTube 검색어 칩 3개” 기능을, UX에서 납득 가능하게 구현한다.
이 기능은 의료 진단/치료가 아니라 “생활 리듬 가이드 + 외부(YouTube) 콘텐츠 추천”이다.
사용자는 복잡한 수치를 보지 않아도 오늘 무엇을 하면 좋을지 이해해야 하며, 추천의 근거(왜)를 확인할 수 있어야 한다.

[절대 원칙(카피/표현)]
- “진단/치료/처방/확정” 표현 금지.
- “~일 수 있어요 / ~해보세요 / 도움이 될 수 있어요” 톤.
- YouTube는 외부 서비스이므로 “외부(YouTube)로 이동합니다.”를 모든 결과 화면에 명시.
- 위험 신호(정서/위기)는 ‘콘텐츠 추천’보다 ‘도움 연결’ 우선(연계 안내 CTA).

====================================================
[추가/확장 컴포넌트]
====================================================
1) CMP/Chip/MetricState
- 형태: pill chip
- 구성: 아이콘 + 라벨 + 상태(좋음/보통/주의)
- 예: “수면 · 보통”, “활동 · 주의”, “피로 · 주의”, “스트레스 · 보통”, “리듬 · 주의”
- 상태는 색만으로 표현 금지(아이콘+라벨 동시)

2) CMP/Tab/RecoCategory
- 4개 카테고리 탭(아이콘+텍스트):
  운동 / 명상 / 식단 / 수면루틴
- 오늘 추천 카테고리에는 “추천” 배지(Accent) 표시
- 탭 전환 시 아래 검색어 칩 3개가 바뀜

3) CMP/Chip/Query
- YouTube 검색어 칩(터치 영역 크게)
- 선택 시 강조(Accent Soft)
- 탭하면 결과 화면으로 이동(검색 실행)
- (옵션) 오른쪽 작은 아이콘 “복사” 제공(길게 누르면 복사)

4) CMP/BottomSheet/WhyReco
- 제목: “왜 이 추천인가요?”
- 상단: MetricState 칩 5개(수면/활동/피로/스트레스/리듬)
- 중단: 한 줄 결론(쉬운 말 + 비유)
- 하단: 오늘 추천 카테고리 + 검색어 칩 3개 다시 노출
- 맨 아래 고정 문구:
  “이 추천은 의료 진단이 아니라, 생활 리듬을 돕기 위한 안내입니다.”

5) CMP/BottomSheet/RecoSettings (선택/최소)
- 제목: “추천 옵션”
- 옵션 2개만 제공(과잉 설정 금지):
  (1) 영상 길이: 5~10분 / 10~15분 / 15~20분
  (2) 강도: 가벼움 / 보통
- 하단 CTA: [적용하기]

6) CMP/State/YouTubeError
- “추천을 불러오지 못했어요” + [다시 시도]
- 오프라인 배너: “인터넷이 약해요. 연결되면 다시 불러올게요.”

====================================================
[규칙: 입력 → 카테고리 선택 → 검색어 칩 3개]
====================================================

(0) 입력값(사용자에게는 ‘상태’로만 보여주기)
- 수면 패턴(워치): 총 수면, 깊은 수면/렘수면, 각성/끊김, 취침/기상 규칙성
- 활동량(워치): 걸음/활동시간
- 피로도(기기 또는 워치 기반): 높음/보통/낮음
- 스트레스(기기 또는 워치 기반): 높음/보통/낮음
- 리듬 안정도(앱 내부 점수 0~100): 낮음/보통/높음

(1) UI에서 보이는 상태(3단계로 단순화)
- 좋음 / 보통 / 주의 3단계로만 표시
- 기준선이 있으면 “최근 7일 평균 대비”로 판정
- 기준선이 없으면 “일반 범위(임상 아님)”로만 완화 표현

(2) 오늘의 추천 카테고리 선택(우선순위 규칙)
아래 우선순위로 ‘오늘 추천 카테고리 1개’를 선택하고 탭에 “추천” 배지를 붙인다.

우선순위 A) 수면루틴 추천 조건(가장 우선)
- 수면이 ‘주의’이거나, 리듬 안정도가 ‘주의’이면 → 오늘 추천 카테고리 = “수면루틴”
(이유: 리듬 회복은 수면/규칙성이 먼저라는 UX 상식 + 사용자 납득이 높음)

우선순위 B) 명상 추천 조건
- 스트레스 ‘주의’ 또는 피로 ‘주의’이고, 수면루틴 조건이 아니면 → “명상”

우선순위 C) 운동 추천 조건
- 활동 ‘주의’이고, 피로가 ‘주의’가 아니면 → “운동”
(피로가 높으면 무리한 운동 대신 회복/명상 우선)

우선순위 D) 식단 추천 조건(유지/안정 day)
- 위 조건(A/B/C)에 걸리지 않고 수면/활동/피로/스트레스/리듬이 전반적으로 ‘좋음/보통’이면 → “식단”
(이유: 좋은 날에는 ‘유지’를 돕는 가벼운 식단/루틴 콘텐츠가 부담이 적음)

(3) 검색어 칩 3개 생성(UX 원칙: 짧게/대안/조금 더)
선택된 카테고리 안에서 3개의 검색어 칩을 생성한다.
- Chip #1: 가장 쉬운/짧은 버전(5~10분, 초보/시니어)
- Chip #2: 같은 목적의 ‘대안’(의자/누워서/짧게 등)
- Chip #3: 조금 더 충분한 버전(10~20분)
사용자의 “추천 옵션(영상 길이/강도)”가 있으면 그에 맞게 템플릿을 선택한다.

(4) “왜 이 추천인가요?” 한 줄 결론 생성(카테고리별 템플릿)
- 수면루틴 선택 시:
  “수면이 조금 들쑥날쑥해서, 오늘은 ‘배터리 충전’부터 도와줄게요.”
- 명상 선택 시:
  “스트레스/피로가 올라가 있어서, 오늘은 ‘긴장 풀기’부터 해보면 좋아요.”
- 운동 선택 시:
  “오늘 활동이 적어서, 가볍게 몸을 깨우는 움직임을 추천해요.”
- 식단 선택 시:
  “오늘은 전반적으로 안정적이라, 부담 없는 식단 아이디어로 리듬을 유지해볼까요?”

(5) 투명성(신뢰) UI
- 모든 추천 화면에 [왜 이 추천인가요?] 링크 제공 → BottomSheet(WhyReco)로 이동
- BottomSheet에는 MetricState 5개 + 한 줄 결론 + 검색어 칩 3개가 다시 보여야 한다.
