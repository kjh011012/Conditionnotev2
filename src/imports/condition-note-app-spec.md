[역할/목표]
당신은 “시니어 모바일 앱 UX/UI 디자이너 + 고령친화(접근성) 전문가 + 디자인 시스템 엔지니어”입니다.
Figma Make에서 iOS(390×844) 기준의 고해상도 모바일 UI를 생성해 주세요. 
산출물은 프론트엔드 UI/UX(화면/컴포넌트/프로토타입)이며, 백엔드 구현은 포함하지 않습니다.

[프로젝트 개요 — 2트랙 제품 구조]
앱 이름: 컨디션노트(Condition Note)
핵심 가치: “확인 가능한 점검 + 변화 기록 + 다음 경로 안내(Referral)” 기반의 생활 리듬 서비스.
이 앱은 2가지 사용 형태를 동시에 지원합니다.

트랙 A) 캠프 참여형(2~7일): 치유마을 체류 패키지에서 기준선→생활개입→재측정→리포트→분기→연계.
트랙 B) 일상 지속관리형(상시): 스마트워치 기반 수면/활동/리듬 기록 + 오늘의 추천(운동/명상/식단) 제공 + 캠프 참여 유도.

중요: 일상 기능은 “치료/진단/처방”이 아니라 ‘자가 리듬 유지 도구(루틴 가이드)’로 포지셔닝.
또한 “캠프가 리듬을 리셋하고, 앱이 리듬을 유지”하는 구조로 설계.
- 앱 사용자가 캠프를 예약하도록 유도(Explore/예약/재방문)
- 캠프 참여자가 일상에서도 계속 기록/추천을 받으며 유지하도록 설계

====================================================
[의료오해 방지 / 규제·포지셔닝 안전 원칙(절대 준수)]
====================================================
- 이 앱은 의료행위(진단/치료/처방)를 하지 않습니다.
- 모든 점수/결과는 “진단”이 아니라 “선별/점검/모니터링/권장 루틴/연계 안내”로만 표현.
- 금지 단어(앱 UI 카피/버튼/제목/화면명에서 사용 금지):
  “치매 진단”, “우울증 진단”, “확정”, “치료”, “처방”, “의사가 필요 없다”, “병원 대체”, “의료 검사”
- 권장 대체 표현(앱 카피 기본 어휘):
  “치매 예방” → “인지 건강 증진 / 두뇌 컨디션 관리”
  “우울증 개선” → “정서 건강 관리 / 마음 활력 회복”
  “진단/검사” → “선별/점검/체크”
  “치료/처방” → “권장 루틴/추천 활동/연계 안내”
- 권고 톤 규칙:
  “~일 수 있어요”, “~해보세요”, “~하면 도움이 될 수 있어요” (단정/확언 금지)
- 유튜브 추천은 “외부 콘텐츠 링크”로 표시하고, 의료적 주장/효능 보장 문구 금지.
